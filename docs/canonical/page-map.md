# Website Page Map

## Purpose

Define the publishable pages for `unifyplane.dev`, their purpose, and the source documents they inherit from.

This page map should stay aligned to the current canonical project documents:

* Problem Statement
* What is UnifyPlane
* Why UnifyPlane Exists
* Website Information Architecture

The page map should support a continuity-first, evidence-led documentation site. It should not drift back into broader or older structures such as generic framework pages, abstract architecture pages, or premature regulatory mapping pages.

---

## Writing (Optional, Secondary)

### Purpose
Provide essays, interpretations, and commentary without disturbing the canonical documentation structure.

**Primary source documents**
* Website Information Architecture
* canonical docs as grounding sources

**Allowed content**
* plain-language explanation of a real enterprise problem
* explanation of one current proven capability
* evidence-backed walkthroughs
* clarifications of common misconceptions
* honest future-direction essays grounded in current canon

**Must not become**
* unsupported thought leadership
* a product funnel
* a faster-moving source of truth than the canonical docs

---

## Page Relationship Rules

### Canonical source pages
These pages define truth and should be reused carefully elsewhere:

* Problem Statement
* What is UnifyPlane
* Current Readiness
* Evidence

### Narrative/supporting pages
These pages should inherit from canonical pages and not redefine them:

* Home
* Why it matters
* Components landing page
* Foundations pages
* Writing

### Rule
If a page summarizes another page’s truth, it should summarize and link, not re-own the same content in different wording unless needed for context.

---

## Pages to Avoid for Now

Do not create these as top-level pages yet:

* model
* frameworks
* architecture
* scenarios
* regulatory mapping
* implementation
* guides

These may become relevant later as secondary or supporting content types, but they are not part of the current continuity-first public structure and would likely dilute the primary signal if introduced too early.
